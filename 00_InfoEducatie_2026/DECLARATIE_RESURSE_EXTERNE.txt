2. Resurse de continut si design

- Iconurile folosite in aplicatie provin in principal din Material Icons / Material Symbols, utilizate conform licentei Google. Nu au fost folosite pachete comerciale sau iconuri fara drept de utilizare.
- Logo-ul si elementele de identitate Liftrix din `assets` si `app/src/main/res/drawable` sunt resurse de proiect ale autorului.
- Aplicatia foloseste in cod fonturile de sistem Android / Material 3 prin `FontFamily.SansSerif`, `FontFamily.Default` si `FontFamily.Monospace`. Exista fisiere font in resurse, dar configuratia Compose activa foloseste fallback-uri de sistem.
- Capturile din `assets/Home-screen.jpg`, `assets/Coach.jpg` si capturile incluse in folderul `00_InfoEducatie_2026/dovezi` sunt capturi ale aplicatiei si pot fi folosite in prezentare.
- Imaginile pentru conturile seed / conturile de test, inclusiv avataruri sau imagini demonstrative asociate acestor conturi, sunt imagini generate cu Gemini Nano Banana 2 si sunt folosite strict ca date demonstrative pentru prezentarea si testarea aplicatiei.
- Imaginile anatomice / imaginile cu muschii folosite in interfata sau in materialele demonstrative sunt imagini generate cu Gemini Nano Banana 2, nu fotografii preluate de pe internet si nu resurse comerciale terte.
- Biblioteca locala de exercitii este stocata in `app/src/main/assets/exercise_library.json` si contine 100 exercitii predefinite. Nu este declarata o sursa externa pentru acest fisier. Denumirile exercitiilor sunt termeni uzuali din domeniul fitness, iar clasificarea pe grupe musculare, echipamente si tipuri de miscare este structurata in proiect de autor.
- Nu au fost utilizate template-uri de proiect platite sau UI kit-uri comerciale terte.

Instrumentele AI au fost folosite ca suport de lucru pentru brainstorming, cercetare, explicatii tehnice, debugging, structurarea documentatiei si sugestii pentru cod repetitiv. Implementarea finala, integrarea in proiect, testarea, deciziile de arhitectura, modelul de date si logica de business au fost realizate si asumate de autor.

Utilizarea AI declarata:

Gemini 2.5 prin Firebase AI / Vertex AI este o functionalitate integrata in aplicatie pentru coaching AI. Aceasta functionalitate este parte din produs si nu inlocuieste contributia autorului la construirea proiectului.

4. Contributia autorului

Autorul a realizat arhitectura aplicatiei, integrarea functionalitatilor, logica de business, modelul de date, fluxurile UI, sistemul offline-first, integrarea Firebase si testarea.

Contributia autorului include:
- Arhitectura: Clean Architecture cu 6 straturi si separare UI / ViewModel / UseCase / Repository / DAO / Firebase.
- Model de date: Room database version 9, cu 69 entitati Room inregistrate in `LiftrixDatabase.kt` si metadata de sincronizare.
- Logic business: 84 use case-uri definite ca clase/interfete in sursa principala, wrapper `LiftrixResult<T>` pentru error handling tipizat si 47 interfete repository in domeniu.
- Servicii de domeniu: 49 interfete service sub domain/service, conform inventarului mentionat in documentatia Word.
- Persistenta si acces la date: 68 DAO-uri/accessors cu filtrare obligatorie per userId.
- Sincronizare: WorkManager, `UnifiedSyncWorker`, workeri specializati si conflict resolution last-write-wins prin timestamp-uri, cu versiunea locala castigand la egalitate in resolverul central.
- Interfata: 61 subclase `LiftrixRoute`, rute/ecrane 30+, componente Compose, sistem de culori LiftrixColorsV2 si integrare Material 3.
- Functionalitati: tracking antrenamente, biblioteca locala cu 100 exercitii predefinite, dashboard de progres cu 12 widget-uri configurabile, social / QR buddy, AI coach si mecanisme offline-first.
- Securitate: user scoping la nivel de DAO, Firebase Security Rules, autentificare, App Check, criptare locala pentru date sensibile si ProGuard/R8 in release.
- Documentare si testare: documentatie tehnica, ghiduri de arhitectura, teste unitare si teste instrumented unde sunt disponibile. Acoperirea procentuala trebuie citata doar dupa rularea JaCoCo.
