### 2. Social Profile Collection Mismatch (7 failures)
**Root Cause:** Architectural inconsistency where social profiles sync to `social_profiles` collection but search queries `users_public`

#### Problem Analysis:
- UserSearchRepository queries `users_public` collection
- SocialProfileSyncWorker syncs data to `social_profiles` collection  
- Privacy settings conflict between UserProfile.isPublic and SocialProfile.isPrivate
- Real-time updates fail due to collection mismatch

#### Fixes Required:

**Fix 1: Standardize on single collection for social search**
```kotlin
// Option A: Update UserSearchRepositoryImpl to query social_profiles
// In app/src/main/java/com/example/liftrix/data/repository/UserSearchRepositoryImpl.kt

private suspend fun searchInFirestore(query: String, currentUserId: String): List<UserSearchResult> {
    return try {
        firestore.collection("social_profiles") // Changed from "users_public"
            .whereGreaterThanOrEqualTo("displayName", query)
            .whereLessThan("displayName", query + '\uf8ff')
            .whereEqualTo("isSearchable", true)
            .get()
            .await()
            .documents
            .mapNotNull { doc -> parseUserSearchResult(doc, currentUserId) }
    } catch (e: Exception) {
        Timber.e(e, "Failed to search users in Firestore")
        emptyList()
    }
}
```
